# Расчет мощностей (Capacity Planning)

## Текущая нагрузка (MVP)
Минимальная нагрузка для демонстрационных целей и PoC.
- **Объем данных**: 100 новостей в сутки.
- **Интенсивность (RPS)**: ~0.001 новости/сек.
- **Ресурсы**: 
  - 1 vCPU
  - 2 GB RAM
  - PostgreSQL 1 GB disk (с запасом ×10 для логов, индексов и дополнительных данных).

## Рост нагрузки: 10 000 новостей в сутки
Сервис начинает активно собирать данные из ключевых бизнес-источников.
- **Интенсивность (RPS)**: ~0.12 новости/сек (в пиках до 1-2 RPS).
- **Архитектура**: Добавление очередей (Redis), разделение на API и фоновые worker-узлы (Arq/Celery).
- **Ресурсы**:
  - 2–4 Worker-инстанса (парсинг и процессинг).
  - PostgreSQL: 4 vCPU / 8 GB RAM / 50 GB disk.
  - Redis: 2 GB RAM.

## Рост нагрузки: 100 000+ новостей в сутки
Полноценный Production с широким охватом и использованием тяжелых ML-моделей.
- **Интенсивность (RPS)**: >1.2 новости/сек (в пиковых нагрузках десятки RPS).
- **Архитектура**: 
  - Замена Redis на надежные брокеры (Kafka/NATS).
  - Sharding или партиционирование таблиц в PostgreSQL.
  - Выделение отдельного NLP-микросервиса (GPU-enabled) для sentiment-анализа через transformers.
  - Autoscaling для worker-узлов (Kubernetes HPA).
- **Ресурсы**:
  - NLP API: GPU-инстансы (T4/A10G), RAM от 16 GB.
  - PostgreSQL: 8+ vCPU / 16+ GB RAM / 500+ GB disk, Read Replicas.
  - Парсинг: Autoscaling pool (до 10-20 подов).
