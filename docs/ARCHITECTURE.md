# Архитектура сервиса (Анализ новостного фона компаний)

## Назначение сервиса
Сервис предназначен для сбора и обработки новостей с российских доменов (.ru), их кластеризации (по компании, локации, отрасли), оценки тональности (sentiment analysis) и агрегации полученного "новостного фона" компаний.

## Стек и обоснование
- **Python (3.12+)**: Богатая экосистема для NLP/ML, скорость разработки MVP/PoC.
- **FastAPI**: Высокая производительность за счет асинхронности (async API), автоматическая генерация OpenAPI спецификаций, встроенная валидация через Pydantic. Подходит под строгие стандарты Clean Architecture, принятые в проекте (слои Domain, Application, Infrastructure, Interfaces).
- **PostgreSQL**: Хранение транзакционных данных. Поддержка JSONB для гибких метаданных и полнотекстовый поиск для быстрого поиска новостей по ключевым словам.
- **Redis (в планах)**: Использование в качестве кэша (быстрые ответы API) и брокера сообщений/задач.
- **Celery/arq (в планах)**: Выполнение фоновых задач, таких как парсинг и агрегация данных.
- **Запуск PoC без Docker**: На этапе MVP запуск осуществляется локально через виртуальное окружение (`venv` / `uv`) для упрощения и ускорения отладки. Контейнеризация будет добавлена на следующем этапе.

## Компоненты системы
1. **Ingestion (Сбор)**: Получение сырых новостей из внешних источников.
2. **Normalization/NER**: Нормализация текста, извлечение сущностей (Названия компаний, Локации, Отрасли).
3. **Clustering (Кластеризация)**: Группировка новостей по нормализованному ключу `(company, location, industry)` с использованием алгоритмов fuzzy-match (например, `rapidfuzz`).
4. **Sentiment (Тональность)**: Определение тональности новости (негативная, нейтральная, позитивная). В PoC используется словарный подход, в дальнейшем — ML-модели на базе transformers (например, `rubert-base-cased-sentiment`).
5. **Storage**: Сохранение результатов анализа в базу данных (PostgreSQL).
6. **API/Reports**: Предоставление доступа к агрегированным данным через REST API и генерация отчетов.

## MVP (Пошаговая реализация)
1. **Модель данных + sample JSON**: Разработка доменных сущностей (`NewsItem`, `Cluster`, `SentimentResult`) и создание тестового датасета (`data/sample_news.json`).
2. **Модули clustering + sentiment**: Реализация логики кластеризации с fuzzy-match и словарного анализа тональности в слое `application/services/`.
3. **CLI demo**: Написание демонстрационного скрипта `app/interfaces/cli/demo.py`.
4. **PostgreSQL-схема**: Подготовка структуры таблиц для хранения новостей и аналитики.
5. **Минимальный FastAPI (опционально для PoC)**: Реализация эндпоинтов `/health`, `/analyze`, `/clusters` в слое `interfaces/api/routes/`.
6. **Тесты**: Написание unit-тестов для проверки работы сервисов кластеризации и тональности (`tests/unit/`).
7. **Документация**: Оформление документации по архитектуре и расчету мощностей (`docs/ARCHITECTURE.md`, `docs/CAPACITY.md`).

## Масштабирование до Production
- Использование брокеров сообщений (Kafka/NATS) для обработки больших объемов данных.
- Горизонтальное масштабирование worker-узлов (Celery/arq).
- Использование Read Replicas в PostgreSQL для разделения нагрузки чтения и записи.
- Внедрение rate limiting для парсеров во избежание блокировок со стороны источников.
- Настройка CI/CD pipeline и контейнеризация (Docker/Kubernetes).
- Разделение NLP задач в отдельные микросервисы.
- Детальный мониторинг и алертинг (Prometheus/Grafana).
